1. Project Overview
The Intelligent CPU Scheduler Simulator is a full-stack educational tool that models and visualizes real CPU scheduling behavior. It implements seven scheduling algorithms — FCFS, SJF, SRTF, Round Robin, Priority Scheduling, Preemptive Priority, and MLFQ — and presents results through real-time Gantt charts, performance metrics, and an AI-powered insights engine.

Goals: Help students deeply understand scheduling mechanics, trade-offs, and algorithm selection through interactive simulation. Expected Outcome: A fully working Python Flask backend paired with a dynamic frontend, hosted on GitHub with proper version control and documentation. Scope: Covers all scheduling topics from CSE250 syllabus, including preemption, starvation, convoy effect, and MLFQ adaptive scheduling.
2. Module-Wise Breakdown
The project is divided into three well-defined modules:

Module 1 — Scheduling Algorithm Engine (Python Backend): This module forms the core computation layer. It implements all seven scheduling algorithms in Python as independent classes, each extending a base Scheduler abstract class. Input is a list of Process objects carrying arrival time, burst time, and priority. Output is a structured ScheduleResult containing the Gantt log, per-process completion/waiting/turnaround times, and aggregate statistics. This module is exposed via a Flask REST API at /api/schedule.

Module 2 — Process Manager (Frontend Input Layer): Handles all process creation, validation, and state management in JavaScript. Users can add/remove processes, adjust parameters, select algorithms, set time quantum, and load preset scenarios. Input validation ensures burst times are ≥1 and arrival times are ≥0. This module communicates with Module 1 by packaging process data as JSON and sending POST requests to the Flask API.

Module 3 — Visualization and AI Insights Engine (Frontend Output Layer): Renders simulation output in five interactive views — Gantt chart with per-process rows and combined CPU timeline, performance metrics dashboard, per-process result table, AI insights with starvation analysis and algorithm recommendations, and an all-algorithm comparison chart. The AI insights layer uses heuristic rules to analyze variance, CPU utilization, fairness, and process mix to recommend optimal algorithms.
3. Functionalities
Module 1 — Algorithm Engine: Implements FCFS (sort by arrival time, sequential execution), SJF (greedy shortest burst selection from ready queue), SRTF (unit-time preemption based on remaining time), Round Robin (circular queue with configurable time quantum), Non-Preemptive Priority, Preemptive Priority (unit-time comparison), and MLFQ (three-level feedback queue: RR q=2, RR q=4, FCFS). All algorithms return completion time, waiting time, turnaround time, and the Gantt timeline log.

Module 2 — Process Manager: Dynamic process table with live-edit inputs for arrival time, burst time, and priority. Process color-coding for visual identification. Preset loader (Basic 5P, Convoy Effect, Starvation scenario, Burst Mix). Time quantum configuration for Round Robin and MLFQ. Reset functionality to restore default state.

Module 3 — Visualization & AI: Animated Gantt chart with per-process rows showing execution windows and idle periods. Average Waiting Time, Average Turnaround Time, CPU Utilization %, and Throughput metrics. Per-process WT/TAT horizontal bar charts. Full result table with completion/waiting/turnaround data. AI insights including fairness index, starvation detection, algorithm recommendation based on process mix, and time/space complexity notes. Side-by-side comparison of all algorithms on the same process set.

7. Conclusion and Future Scope
The IntelliSched CPU Scheduler Simulator successfully demonstrates all major scheduling algorithms covered in the CSE250 Operating Systems course. It bridges theoretical concepts with hands-on visual understanding, making it a valuable learning tool. The project integrates previous semester learnings including Python programming, web technologies (HTML/CSS/JS), REST API design, data structures (queues, lists), and algorithm design.

Future Scope: (1) Real-time OS process integration using the Python psutil library to schedule actual running processes. (2) Machine learning module to predict optimal algorithm based on historical process patterns. (3) Multi-core CPU simulation with processor affinity. (4) Export feature to generate PDF reports of simulation results. (5) Mobile-responsive PWA version deployable on smartphones.
